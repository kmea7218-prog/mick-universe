# Workflow: 產出月報 / 週報

## 觸發
- 月初(月報)
- 每週一(週報)
- 客戶臨時要求

## 步驟
1. **確認期間**與範圍(哪個牧場、哪些 KPI)
2. 從 `data/farms/<farm>/standardized/` 撈資料
3. 算 KPI(data-analyst),結果寫成 `data/reports/<farm>/<yyyy-mm>_monthly.md`
4. **對比**:與上個月、去年同期、同規模牧場匿名平均(若有)
5. **亮點 + 警訊**:每項至多 3 個,不要每個數字都講
6. **建議**:1–3 個具體可做的動作
7. 輸出 PDF / Markdown 兩版;Markdown 進 git,PDF 給客戶
8. 交付前人工複查(不自動寄出)

## 報告結構(模板)
```
# <牧場名> 月報 / <yyyy-mm>

## 本月重點(3 點內)

## KPI 總覽
- 繁殖
- 產乳
- 健康
- 營運

## 亮點 / 警訊

## 下月建議動作

## 資料說明
- 資料期間:
- 牛隻數:
- 缺資料:
```
