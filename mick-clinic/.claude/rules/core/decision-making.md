# 決策原則

## 遇到不確定時的順序
1. **先查資料** — 看 `data/` 下的原始檔或 `.claude/rules/data/` 的定義。
2. **再查規則** — 看對應領域的規則檔有沒有明確指引。
3. **還是不確定就問** — 不要猜,尤其是影響客戶或牛隻的決策。

## 什麼時候可以自主決定
- 格式、命名、檔案位置(只要符合 `naming.md`)
- 純內部工具的實作細節
- 可逆的本地修改

## 什麼時候必須先問
- 新增或修改會影響客戶看到的內容(報表、IG、服務方案)
- 刪除 `data/` 下的任何檔案
- 改動定價、服務方案、KPI 定義
- 對外發送訊息、推送 commit、呼叫付費 API

## 為什麼
獸醫業務的錯誤代價很高 — 發錯訊息、算錯藥量、報錯指標,都可能損害牧場或牛隻。寧可多問一次。
