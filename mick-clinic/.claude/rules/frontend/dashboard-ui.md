# Dashboard UI 設計原則

## 使用者
主要是**牧場主**與**獸醫**,不是資料分析師。他們要在手機或牛舍旁快速讀懂。

## 原則
1. **一眼一個重點** — 每個區塊只強調一個數字或趨勢。
2. **顏色只有三種意義**:正常(灰/綠)、需注意(黃)、警示(紅)。不要用顏色當裝飾。
3. **單位永遠可見** — `21 天`、`305 kg`、`65%`,不要讓使用者猜。
4. **缺值明顯標示**:用 `—` 或「無資料」,不要顯示 0 或空白。
5. **預設時間範圍**:30 天;重要指標同時顯示去年同期。

## 互動
- 點數字可看原始資料或定義(連到 `.claude/rules/data/kpi-definition.md`)
- 所有圖表都能下載 PNG 與 CSV

## 技術棧
Next.js(已初始化於 `mick-universe/`)+ Tailwind + 圖表庫待定(建議 Recharts 或 Visx)。
