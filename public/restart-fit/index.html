技術架構 · MVP
產品定位

「不是讓你變完美，而是幫你重新開始。」AI 健身習慣養成 App，主打陪伴型教練、低壓力、有最低完成版。

五大頁面
  1. 今日 — 訓練 + 飲食 + 累時切換
  2. 訓練 — 紀錄重量／次數／RPE
  3. 飲食 — 拍照辨識熱量／巨量
  4. 進度 — 體重、連續天數、完成率
  5. AI 教練 — 自由問答對話
資料庫 schema（Supabase）
users               -- 基本資料
  id, email, name, height_cm, weight_kg, age, gender,
  goal, weekly_days, session_minutes, equipment[],
  fitness_state, created_at

workouts            -- 一次訓練 (header)
  id, user_id, date, plan_phase, total_minutes,
  completion_pct, mode (full|low|skipped), notes

exercises           -- 動作字典
  id, name, category (push|pull|legs|core|cardio),
  primary_muscle, equipment, demo_video_url

workout_logs        -- 單一動作的所有組
  id, workout_id, exercise_id, set_no, reps,
  weight_kg, rpe, completed, note

meals               -- 一餐 (header)
  id, user_id, datetime, photo_url, meal_type

meal_analysis       -- AI 拍照辨識結果
  id, meal_id, items_json, kcal, protein_g,
  carb_g, fat_g, ai_feedback

ai_feedback         -- 每日 AI 回饋
  id, user_id, date, kind (workout|meal|day),
  content, mood_tag

progress_tracking   -- 每週快照
  id, user_id, week, weight_kg, body_fat_pct,
  streak_days, completion_rate

API 路由
  • POST /api/workout — 記錄一組
  • POST /api/meal — 上傳照片 → Vision 辨識
  • POST /api/coach/chat — Claude 對話
  • POST /api/coach/daily-feedback — 每日總結
  • GET /api/plan/today — 今日課表
  • POST /api/plan/regenerate — 根據完成率調整
AI Workflow
陪伴語氣 system prompt：禁止責備／壓迫；鼓勵維持。
每日上下文注入：今日課表 + 過去 7 天完成率 + 最近 3 餐 + 主觀疲勞。
飲食拍照：GPT-4o Vision → 食物項目 + 熱量估算（不追求 100% 精準）。
課表自動調整：完成率 < 60% → 下週降量 15%；> 90% 連 2 週 → 升量 10%。
技術棧
  • 前端：Next.js 14 + Tailwind（PWA）／React Native（v2）
  • 後端：Supabase（auth + DB + storage）
  • AI：Anthropic Claude（對話）＋ OpenAI GPT-4o Vision（拍照）
  • 分析：Vercel + simple cron 每週快照
擴充路線圖
  • v1 MVP — 五頁面、最低完成版、AI 對話、拍照記食
  • v1.1 — 連續天數、徽章、回顧週報
  • v1.2 — Apple Health／Google Fit 同步
  • v2 — 朋友圈、組隊重啟、教練模式（人類教練 dashboard）
  • v3 — Wearable 即時心率、智能熱身建議
